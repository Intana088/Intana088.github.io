  Software Requirement Inspection Laboratory
 Introduction:
                 Software inspection is a proven approach for detecting and removing defects immediately after software documents are created[Oliver and Khaled, 2000]. Inspectors use reading techniques for inspecting the SRS. A software reading technique provides each reader a well-defined process that explains how to read a software document, written in a specific notation, and what the reviewer should look for. Currently techniques include[Guzm?n and Nemer, ????]: 
                (1) Ad hoc, an unstructured review which provides no guidance, based only on individual knowledge and experience; 
                (2) Checklist provides readers a list of issues and questions to focus on, capturing the knowledge of previous inspections; and 
                (3) Scenario-based approach requires the reader to create first a model of the product and then to answer questions based
                      on  analyzing the model with a particular emphasis. In this context, a scenario is a script or procedure that the reviewer should 
                      follow.
