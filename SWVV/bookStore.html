  BookStore Case Study

 

 

 

Requirments – User Stories

  

Title: Waiting State

AccTest: checkOptions

Priority: 1

  

When the BookStore System is not in use it waits for user input. There are six different options  of user input: 1) adding an item into shopping cart, 2) viewing items from shopping cart, 3) deleting an item in a shopping cart, 4) deleting a shopping cart, 5) purchasing a shopping cart, and 6) exit the system

 

Title: Adding an item into shopping cart

AccTest: addItem

Priority: 1

  

Only four shopping carts may be added to the shopping cart. A shopping cart consists of  a customer data (customer id, customer name, customer address,  customer type), order date and order details (consists of product name, product author and price). Price must be handled as an integer or double. If there are some error in the system, a status message is printed. The BookStore System is returned to the waiting state.

  

Title: Viewing items from shopping cart

AccTest: viewItems

Priority: 2

  

A list of items in the desired shopping cart are displayed. The user will be choosed the shopping cart number to display the items in it.

  

Title: Deleting an item from shopping cart

AccTest: deleteItem

Priority: 2

  

The shopping cart items may be deleted from its shopping cart, if it exists in the list of shopping carts in the BookStore. An item to be deleted is chosen by its number. If there are some errors in the system, a status message is printed. The BookStore System is returned to the waiting state.

  

Title: Deleting a shopping cart

AccTest: deleteShoppingCart

Priority: 2

  

A shopping cart may be deleted, if it exists in the list of shopping carts in the BookStore. A shopping cart to be deleted is chosen by its number. If there are some errors in the system, a status message is printed. The BookStore System is returned to the waiting state.

 

  

Title: Purchasing a shopping cart

AccTest: purchasing

Priority: 1

  

The user will not be able to purchase a shopping cart if they do not deposit enough money to the BookStore. If they pay more than the price of the shopping cart, Change will be dispensed to the user. If  there is not enough money to pay, A user’s money will be returned. The BookStore is returned to the waiting state

 

 

 

Business Rule--Purchasing

If customer is a member, and

            If customer order less than ฿1000 then

                        Apply a 3% discount

            else if customer order is greater than and equal  ฿1000 and less than ฿2000 then

                        Apply a 5% discount

                else if customer order is greater than and equal  ฿2000 and less than ฿3000 then

                        Apply a 7% discount

            else if customer order is greater than and equal  ฿3000 and less than ฿4000 then

                        Apply a 10% discount

            else if customer order is greater than and equal  ฿4000 and less than ฿5000 then

                        Apply a 15% discount

            else

                        Apply a 20%discount

            End if

 

            If customer uses our bronze card then

                        Apply an additional 3% discount

           else if customer uses our silver card then

                        Apply an additional 5% discount

            else if customer uses our gold card then

                        Apply an additional 10% discount

            End if

End if

 

If customer uses bonus coupon then

            Appy an additional coupon discount [accept only 5%-30%]

End if

 

Requirement – Use Cases

  

  

  

UC1: Flow of Events for Waiting State Use Case

1.1  Preconditions: The system displays the main menu.

1.2  Main Flow: The BookStore waits for user input. There are six options to choose from [E1]:

1)      adding an item to shopping cart [UC2],

2)      viewing items from shopping cart[UC4],

3)      deleting an item from shopping cart [UC5],

4)      deleting a shopping cart [UC6],

5)      purchasing a shopping cart [UC7],

6)      and exit the system [UC8].

1.3  Postcondition: The BookStore loads the operation according to the chosen option

 

UC2: Flow of Events for Adding Shopping Cart Use Case

2.1  Preconditions: None

2.2  Main Flow:  To create the shopping cart [E1], user needs to fill the customer id who is the owner of this shopping cart to retrive customer information [E2] which include the customer’s name, customer’s address and customer’s type (There are 3 types: Bronze member, Silver member and Goal member). The order date will be generated by the system.  The user adds shopping items into his created shopping cart.[UC3]

2.3  Sub Flow: None

2.4  Alternative Flows:

[E1] if there are four shopping carts in the system, the system will reject creating new shopping cart. The user is returned to the main menu.

[E2] if the system can not find that customer id (He is not the member of the BookStore), the system will replace customer id with 999 (He is walk in customer).

2.5  Postconditions: The system displays the screen for adding the shopping cart item.

 

UC3: Flow of Events for Adding Shopping Cart Items Use Case.

3.1    Preconditions: None

3.2    Main Flow: The user fills the shopping item which include the book name, book author, book price [E1] [E2].

3.3    Sub Flow: None

3.4    Alternative Flows:

[E1] The book price must be integer or double. If the book price is not integer or double then, a status message is printed, and the user is returned to the main menu.

[E2] If the book price is negative, a message will be displayed and the user will be returned to the main menu.

3.5    Postconditions: The system displays the main menu.

 

UC4: Flow of Events for Viewing the Shopping Cart Details.

4.1    Precondition: The system  shows a list of all shopping carts in the system

4.2    Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to view.[E1] All item of the selected shopping cart are listed [E2].

4.3    Sub Flow: None

4.4    Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] If there is no item in the shopping cart, the system will display blank record on the screen.

4.5    Postconditions: The system displays the main menu

 

 UC5: Flow of Events for Deleting an  Item from Shopping Cart Use Case.

5.1  Precondition : The system  shows a list of all shopping carts in the system

5.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to delete.[E1] All item of the selected shopping cart are listed for letting the user to delete the desired item.[E2]

5.3  Sub Flows:

5.4  Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] If the user selects an shopping item number that is out of bounds of the number of  items, a status message is printed, and the user is returned to the main menu.

5.5  Postcondition: The system displays a status message and wait for the user to press any key to return to the main menu.

 

UC6: Flow of Events for Deleting a Shopping Cart Use Case.

6.1  Precondition : The system  shows a list of all shopping carts in the system

6.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to delete.[E1]

6.3  Sub Flows:

6.4  Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

6.5  Postcondition: The system displays a status message and wait for the user to press any key to return to the main menu.

 

UC7: Flow Events for Purchasing the Shopping Cart.

7.1  Precondition : The system will show a list of all shopping carts in the system

7.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to purchase [S1] [E1]. The user will enter the coupon discount [E2][E3] and  deposit (or enter) money [E4] to pay for the shopping cart.[S2]

7.3  Sub Flow:

[S1] The total price of the shopping cart will be displayed.

[S2] The system will make sure enough money was deposited [E5], any extra change will be returned.

7.4  Alternative Flow:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] The coupon discount must be in range 5-30%. If the coupon discount is out of ranges, a status message will be displayed and the user will be entered coupon discount again.

[E3] If the coupon discount is negative, a message will be displayed and the user will be entered coupon discount again.

[E4] If the deposit money is negative, a message will be displayed and the user will be entered money again.

[E5] If the user does not enter enough money, a message will be displayed and the user will be entered money again..

7.5  Postcondition:  The system displays the change.
