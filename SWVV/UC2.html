Requirments – User Stories

 

 

Title: Waiting State

AccTest: checkOptions0

Priority: 1

 

Whe the Shopping Cart System is not in use it waits for user input. Threr are six different options  of user input: 1) add an item into shopping cart, 2) view items from shopping cart, 3) delete an item in a shopping cart, 4) delete a shopping cart, 5) purchase a shopping cart, and 6) exit the system

 

Title: Add an item into shopping cart

AccTest: addShoppingCart1

Priority: 1

 

Only four shopping carts may be added to the shopping cart. A shopping cart consists of  a customer data (customer id, customer name, customer address,  customer type), order date and order details (consists of product name, product author and price). Price must be handled as an integer. If there is some error in the system, an status message is printed. The BookStore System is returned to the waiting state.

 

Title: View items from shopping cart

AccTest:

Priority: 2

 

A shopping cart items may be displayed. The user will be choosed the shopping cart number to display the items in it.

 

Title: Delete an item from shopping cart

AccTest:

Priority: 2

 

A shopping cart items may be deleted from its shopping cart, if it exists in the list of shopping carts in the BookStore. An item is to be deleted is chosen by its number. If there is some error in the system, an status message is printed. The BookStore System is returned to the waiting state.

 

Title: Delete a shopping cart

AccTest:

Priority: 2

 

A shopping cart may be deleted, if it exists in the list of shopping carts in the BookStore. A shopping cart is to be deleted is chosen by its number. If there is some error in the system, an status message is printed. The BookStore System is returned to the waiting state.

 

Title: Purchase a shopping cart

AccTest:

Priority: 1

 

The user will not be able to purchase a shopping cart if they do not deposit enough money into the BookStore. If they paid more than the price of the shopping cart, Change will be deispensed to the user. If  there is not enough money to pay, A user’s money will be returned. The BookStore is returned to the waiting state

 

 

 

 

 

 

UC1: Flow of Events for Waiting State Use Case

1.1  Preconditions: The system displays the main menu.

1.2  Main Flow: The BookStore waits for user input. There are six options to choose from [E1]:

1)      add an item to shopping cart [UC2],

2)      view items from shopping cart[UC4],

3)      delete an item from shopping cart [UC5],

4)      delete a shopping cart [UC6],

5)      purchase a shopping cart [UC7],

6)      and exit the system [UC8].

1.3  Postcondition: The BookStore loads the operation according to the chosen option

 

UC2: Flow of Events for Adding an Item to Shopping Cart Use Case

2.1  Preconditions: None

2.2  Main Flow:  To create the shopping cart, user needs to fill the customer id who is the owner of this shopping cart to retrive customer information [E1] which include the customer’s name, customer’s address. The order date will be generated by the system. [E2].  The user adds shopping items into his created shopping cart.[UC3]

2.3  Sub Flow: None

2.4  Alternative Flows:

[E1] if the system can not find that customer id (He is not the member of the BookStore), the system will replace customer id with 999 (He is walk in customer)

[E2] if there are four shopping carts in the system, the system will reject creating new shopping cart. The user is returned to the main menu.

2.5  Postconditions: The system displays the screen for adding the shopping cart item.

 

UC3: Flow of Events for Add A Shopping Cart Item Use Case.

3.1    Preconditions: None

3.2    Main Flow: The user fills the shopping item which include the book name, book author, book price [E1].

3.3    Sub Flow: None

3.4    Alternative Flows:

[E1] The book price must be double. If the book price is not double then, a status message is printed, and the user is returned to the main menu.

3.5    Postconditions: The system displays the main menu.

 

UC4: Flow of Events for Viewing the Shopping Cart Details.

4.1    Precondition: The system  shows a list of all shopping carts in the system

4.2    Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to view.[E1] All item of the selected shopping cart are listed [E2].

4.3    Sub Flow: None

4.4    Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] If there is no item in the shopping cart, the system will display blank record on the screen.

4.5    Postconditions: The system displays the main menu

UC5: Flow of Events for Deleting an  Item from Shopping Cart Use Case.

5.1  Precondition : The system  shows a list of all shopping carts in the system

5.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to delete.[E1] All item of the selected shopping cart are listed for letting the user to delete the desired item.[E2]

5.3  Sub Flows:

5.4  Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] If the user selects an shopping item number that is out of bounds of the number of  items, a status message is printed, and the user is returned to the main menu.

5.5  Postcondition: The system displays a status message and wait for the user to press any key to return to the main menu.

 

UC6: Flow of Events for Deleting a Shopping Cart Use Case.

6.1  Precondition : The system  shows a list of all shopping carts in the system

6.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to delete.[E1]

6.3  Sub Flows:

6.4  Alternative Flows:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

6.5  Postcondition: The system displays a status message and wait for the user to press any key to return to the main menu.

 

UC7: Flow Events for Purchasing the Shopping Cart.

7.1  Precondition : The system will show a list of all shopping carts in the system

7.2  Main Flow: Shopping carts in the system are shown. The user selects the shopping cart that contains some items he would like to purchase. [S1] [E1]  The user will deposit (or enter) money [E2] to pay for the shopping cart.[S2]

7.3  Sub Flow:

[S1] The total price of the shopping cart will be displayed.

[S2] The system will make sure enough money was deposited [E2], any extra change will be returned.

7.4  Alternative Flow:

[E1]  If the user selects a shopping cart number that is out of bounds of the number of shopping carts, a status message is printed, and the user is returned to the main menu.

[E2] If the deposit money is negative, a message will be displayed and the user will be returned to the main menu.

[E2] If the user does not enter enough money, they will be prompted for more money or given the option to return to the main menu and have their money returned.

7.5  Postcondition:  The system displays the change.
